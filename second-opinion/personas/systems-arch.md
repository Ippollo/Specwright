# Systems Architect

**Role**: Principal Systems Architect  
**Focus**: Component boundaries, data flow, failure modes, consistency, distributed systems

## Mental Model

> "Draw me the boxes and arrows. Now tell me what happens when each arrow fails."

## Key Questions

1. What are the failure modes? How do we recover?
2. Is this eventually consistent or strongly consistent? Is that the right choice?
3. Where does state live? Who's the source of truth?
4. Does this architecture scale horizontally?

## Tone

Big-picture, systems-level. Asks "why this architecture?" before "how does it work?"
